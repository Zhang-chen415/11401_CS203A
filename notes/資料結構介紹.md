# Data Structures (資料結構) 介紹

資料結構（Data Structure） 是電腦組織、管理與儲存資料的一種特定方式，目的是為了讓資料能被有效率地存取、修改與運算。

## 為什麼資料結構很重要？
- 效率（Efficiency）：選擇正確的資料結構可以大幅提升程式執行速度。
- 記憶體管理（Memory Management）：有效的組織能減少不必要的記憶體浪費。
- 可重用性（Reusability）：建立標準化的資料處理方式，便於維護與擴充。

## 資料結構的分類
資料結構主要可以分為以下兩大類：

## 1. 線性資料結構
資料元素呈線性排列，每個元素（除了第一個和最後一個）都有唯一的前驅和後繼。

### 1.1 Array (陣列)
* **原理**：在記憶體中使用一塊**連續**的空間。
* **優點**：可透過索引 (Index) 在 $O(1)$ 時間內隨機存取。
* **缺點**：插入與刪除需要搬移資料 ($O(n)$)，且大小通常固定。



### 1.2 Linked List (鏈結串列)
* **原理**：由「節點 (Node)」組成，每個節點包含資料與指向下個節點的「指標 (Pointer)」。
* **分類**：單向串列、雙向串列、環狀串列。
* **應用**：動態記憶體配置、實作 Stack 或 Queue。

### 1.3 Stack (堆疊) 與 Queue (佇列)
* **Stack**：**後進先出 (LIFO)**。應用於函式呼叫堆疊、回溯 功能。
* **Queue**：**先進先出 (FIFO)**。應用於任務排程、廣度優先搜尋 (BFS)。



---

## 2. 非線性資料結構

資料元素之間存在一對多或多對多的層級或網路關係。

### 2.1 Tree (樹)
樹狀結構由根節點 (Root) 出發，常用於表現階層關係。
* **Binary Search Tree (BST)**：左子節點 < 父節點 < 右子節點。
* **AVL / Red-Black Tree**：自動平衡樹，確保搜尋效率維持在 $O(\log n)$。
* **Heap (堆積)**：常用於實作「優先權佇列 (Priority Queue)」。



### 2.2 Hash Table (雜湊表)
* **原理**：透過**雜湊函數 (Hash Function)** 將 Key 對應到特定的 Bucket 位置。
* **碰撞處理 (Collision)**：
    * **Chaining**：在同位置使用 Linked List。
    * **Open Addressing**：尋找下一個空位置。



### 2.3 Graph (圖)
由節點 (Vertex) 與邊 (Edge) 組成的網路，是最強大且複雜的結構。
* **表示方式**：相鄰矩陣、相鄰串列。
* **應用**：社交網路分析、地圖導航、網路爬蟲。

## 各類資料結構效能比較表

| 資料結構 | 搜尋 (Search) | 插入 (Insertion) | 刪除 (Deletion) | 常用場景 |
| :--- | :--- | :--- | :--- | :--- |
| **Array** | $O(n)$ / $O(1)$ | $O(n)$ | $O(n)$ | 已知數量、讀取頻繁 |
| **Linked List** | $O(n)$ | $O(1)$ | $O(1)$ | 頻繁插入刪除、動態大小 |
| **BST** | $O(\log n)$ | $O(\log n)$ | $O(\log n)$ | 排序資料管理 |
| **Hash Table** | $O(1)$ | $O(1)$ | $O(1)$ | 字典、快速鍵值搜尋 |
